# 4. Where DBT Fits

This is the most important concept.

DBT **only handles transformation**.

DBT does NOT do:

❌ data extraction
❌ data ingestion

DBT does:

```
SQL transformations
```

Example:

Raw table:

```
raw.ratings
```

DBT model:

```sql
SELECT movie_id,
       AVG(rating) AS avg_rating
FROM raw.ratings
GROUP BY movie_id
```

DBT creates table:

```
analytics.movie_ratings
```

So DBT = **SQL transformation engine**.

# 6. ETL vs ELT (Super Important)

Old architecture:

### ETL

```
Extract
Transform
Load
```

Example:

```
source
   ↓
transform in Spark
   ↓
load into warehouse
```

Transformation happens **before warehouse**.

---

Modern architecture:

### ELT

```
Extract
Load
Transform
```

Example:

```
source
   ↓
load raw data into warehouse
   ↓
transform inside warehouse
```

Transformation happens **inside warehouse**.

DBT works with:

```
ELT
```

---

# 7. Why ELT Became Popular

Because modern warehouses like:

• Snowflake
• Google BigQuery
• Amazon Redshift

are **extremely powerful**.

They can process huge data directly.

So instead of using Spark pipelines:

```
just load raw data
then run SQL transformations
```

This is exactly what DBT does.
