19.Sources 

---

# 1. What is “Source” (actual concept)

### Simple meaning:

**Source = raw data already present in your database**

You are just telling DBT:

> “Hey, this table exists, I will use it”

---

## Example (your project)

You already have tables in Snowflake:

```text
raw.movies
raw.genome_scores
raw.genome_tags
```

These came from S3 → Snowflake

---

### Without source

You write:

```sql
SELECT * FROM raw.movies
```

Works fine.

---

### With source

You define in YAML:

```yaml
sources:
  - name: netflix
    schema: raw
    tables:
      - name: movies
```

Then use:

```sql
SELECT * FROM {{ source('netflix', 'movies') }}
```

---

## SAME RESULT

But better structure.

---

# 2. Why sources exist (actual purpose)

### 1. Documentation

```text
Where data came from?
→ Netflix raw schema
```

---

### 2. Testing

You can check:

* null values
* duplicates

---

### 3. Freshness (very important in real jobs)

```text
Is data updated or stale?
```

Example:

* If no new data in 2 days → alert

---

### 4. Standard naming

Instead of messy:

```sql
raw.movies
raw_movies
movies_raw
```

You use:

```sql
source('netflix','movies')
```

Clean + consistent

---

# 3. Real-life analogy

* Source = **raw ingredients**
* Model = **cooking**
* Final table = **dish**

---

# 4. Difference from others (very simple)

| Thing  | What it does           |
| ------ | ---------------------- |
| Source | Uses existing raw data |
| Seed   | Uploads CSV as table   |
| Model  | Transforms data        |


1. What is this .yml file?
Simple meaning:

YAML file = configuration + metadata (info about your data)

It does NOT process data
It just tells DBT how to understand data

2. What it actually does

This file tells DBT:

where data comes from (source)

what tables exist

what columns mean

what rules to check (tests)

how fresh data should be

3. Your example (simple version)
version: 2

sources:
  - name: netflix
    schema: raw
    tables:
      - name: movies
What this means
name: netflix  → group name (you choose)
schema: raw    → where table exists
table: movies  → actual table name
How you use it
SELECT * FROM {{ source('netflix', 'movies') }}
4. Now adding REAL power (this is important)
Full example
version: 2

sources:
  - name: netflix
    schema: raw

    tables:
      - name: movies
        description: "Raw movies data from S3"

        columns:
          - name: movie_id
            description: "Primary key"

            tests:
              - unique
              - not_null

          - name: title
            description: "Movie name"

    freshness:
      warn_after: {count: 1, period: day}

5. What each thing does

1. Description
Helps documentation

In DBT docs UI → shows meaning of table/column

2. Tests
tests:
  - unique
  - not_null

DBT checks:

no duplicates

no nulls

If fails → error

3. Freshness
warn_after: 1 day

Means:

If data not updated in 1 day → warning
