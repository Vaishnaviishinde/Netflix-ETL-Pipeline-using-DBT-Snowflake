2.DBT components 

Here are the **6 important DBT concepts in short and simple form**.

---

# 1. Models

A **model is a SQL file** in DBT used to transform data.

Location:

```
models/
```

Example file:

```
models/movie_ratings.sql
```

Example SQL:

```sql
SELECT movie_id,
       AVG(rating) AS avg_rating
FROM raw.ratings
GROUP BY movie_id
```

When you run DBT, it creates a table or view in Snowflake.

Purpose:
Transform raw data into clean analytics tables.

---

# 2. ref()

`ref()` is used to **reference another DBT model**.

Example:

Model 1:

```sql
SELECT *
FROM raw.movies
```

Model 2:

```sql
SELECT *
FROM {{ ref('movies') }}
```

Purpose:

* Connect models
* Create dependency order
* DBT knows which model runs first

---

# 3. Seeds

Seeds allow you to **load CSV files as tables**.

Location:

```
seeds/
```

Example file:

```
genres.csv
```

Command:

```
dbt seed
```

DBT creates a table in Snowflake.

Purpose:
Upload small static datasets.

Examples:

* country codes
* lookup tables
* category mappings

---

# 4. Snapshots

Snapshots **track changes in data over time**.

Example:

User subscription status changes:

```
user_id | status
1       | active
1       | cancelled
```

Snapshot stores history of these changes.

Purpose:
Maintain historical data.

Example use cases:

* subscription history
* price changes
* customer status tracking

---

# 5. Macros

Macros are **reusable SQL functions**.

Example macro:

```sql
{% macro calculate_revenue(price, quantity) %}
    {{ price }} * {{ quantity }}
{% endmacro %}
```

Use in model:

```sql
SELECT
    {{ calculate_revenue(price, quantity) }} AS revenue
FROM sales
```

Purpose:
Avoid repeating SQL logic.

---

# 6. Tests

Tests check **data quality**.

Example tests:

* column should not be null
* values should be unique

Example test:

```yaml
tests:
  - not_null
  - unique
```

Purpose:
Ensure data is correct before analytics.

---

# One-line summary

Models transform data, `ref()` links models, seeds load CSV data, snapshots track history, macros reuse SQL logic, and tests validate data quality in dbt (Data Build Tool).
