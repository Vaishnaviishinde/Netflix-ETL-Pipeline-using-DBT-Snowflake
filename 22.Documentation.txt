22.Documentation 

---

## 1. What is DBT Documentation?

DBT automatically creates a **website for your data project**
This website shows:

* tables (models, sources, snapshots)
* columns + descriptions
* tests applied
* SQL logic
* lineage (data flow)

👉 Think of it like: **Google Maps for your data pipeline**

---

## 2. Commands (VERY IMPORTANT)

### 1. Generate docs

```bash
dbt docs generate
```

**What it does:**

* Reads all `.sql` + `.yml` files
* Collects:

  * models
  * sources
  * tests
  * descriptions
* Builds metadata (JSON files internally)

---

### 2. View docs

```bash
dbt docs serve
```

**What it does:**

* Starts a **local web server**
* Opens browser (localhost)
* Shows full interactive documentation

---

## 3. Where does this data come from?

Mainly from:

* `schema.yml` → descriptions + tests
* `sources.yml` → raw data info
* SQL models → transformation logic

---

## 4. What you see in DBT Docs UI

### A. Models / Tables

Example:

* dim_movies
* fct_ratings

You can click and see:

* columns
* descriptions
* SQL query
* tests

---

### B. Columns info

Example:

* movie_id → “primary key”
* movie_title → “standardized name”

👉 Comes from YAML descriptions

---

### C. Tests

You’ll see:

* not_null
* unique
* relationships

👉 Helps understand data quality rules

---

### D. SQL Logic

You can see:

* full query used to build table
  👉 very useful for debugging

---

### E. Lineage Graph (MOST IMPORTANT)

This shows:

* how tables are connected
* which table depends on which

Example flow:

```
raw_movies → src_movies → dim_movies → mart_movie
```

👉 One glance = full pipeline understanding

## 6. Common Mistakes (he kinda skipped clarity)

### ❌ Wrong:

```sql
sources('netflix','movies')
```

### ✅ Correct:

```sql
source('netflix','movies')
```

---

### ❌ Not writing descriptions

→ docs become useless

---

### ❌ Not running generate before serve

```bash
dbt docs serve   ❌
```

```bash
dbt docs generate
dbt docs serve   ✅
```
